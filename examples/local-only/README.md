# Local-only examples — runs free on your laptop

These templates use [Ollama](https://ollama.com) for every role so you
get a real multi-agent debate with **zero cloud cost and zero keys**.

## One-time setup

```bash
# 1. Install Ollama
#    macOS:        brew install ollama
#    Linux:        curl -fsSL https://ollama.com/install.sh | sh
#    Windows:      https://ollama.com/download/OllamaSetup.exe

# 2. Pull a model. llama3.1:8b is the demo default (about 5 GB).
ollama pull llama3.1:8b

# 3. Install the Orchestra adapter extra (LiteLLM speaks Ollama).
pip install 'grok-agent-orchestra[adapters]'

# 4. Confirm everything's live.
grok-orchestra doctor
```

`doctor` should report **✅ Local mode ready** with `llama3.1:8b` in the
detected models list.

## Run

```bash
grok-orchestra run examples/local-only/local-research.yaml
```

You'll see the same Grok / Harper / Benjamin / Lucas debate streamed
into the Rich TUI — only every token is generated by Ollama on your
machine. The run summary tags `mode_label="adapter"` and
`provider_costs={}` (no cloud invoice).

## What this is **not**

- Not as good as Grok / Claude / GPT-4. `llama3.1:8b` is solid for
  drafts, not for ship-grade output. The framework's value
  proposition (visible debate + Lucas veto) still applies — the
  reasoning quality scales with the model.
- Not network-isolated by default. The Ollama server itself runs
  locally, but if you uncomment the `sources:` block in
  `local-research.yaml` you'll need a Tavily key (BYOK). Leaving
  `sources:` commented keeps the run fully offline.

## Three-tier path

```
Demo mode    →  --dry-run, no setup, pre-canned events
Local mode   →  Ollama, real LLM, free, this directory
Cloud mode   →  Grok / OpenAI / Anthropic, BYOK, ship-grade
```

`grok-orchestra doctor` tells you which tiers your machine has
configured right now.

## Where to from here

- Try a different local model: change `ollama/llama3.1:8b` to
  `ollama/qwen2.5:7b` or `ollama/phi3:mini` and re-run. Anything
  Ollama can run, the framework can drive.
- Mix tiers: leave Harper / Benjamin / Grok on Ollama but swap Lucas
  to `xai/grok-2-latest` (BYOK) for a stricter veto. The dispatcher
  detects the mixed config and tags the run `mode_label="mixed"`.
- Ship to production: re-run with cloud models. The YAML is portable
  — only the `model:` strings change.
